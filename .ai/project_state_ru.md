# Состояние проекта

## Текущий прогресс

- **v1 (backend_snt_core):** Реализован и стабилен. PostgreSQL, документы, проводки, `reg_balance`, REST API, seed. UI только Swagger / простая главная.
- **v2 (backend_snt_v2):** Реализован. SQLite, справочники (физ. лица, СНТ, члены, участки, счётчики, пользователи), REST API, авторизация через `X-User-Id`, строгая изоляция по СНТ для `snt_user`.
- **Frontend:** Реализован. Использует только v2 API. Выбор пользователя и СНТ, дашборд, таблицы физ. лиц / членов / участков / счётчиков, модалка деталей физ. лица. Только просмотр.

---

## Реализовано

| Область | Детали |
|---------|--------|
| **v2 API** | `GET /users`, `/me`, `/snts`, `/physical-persons`, `/physical-persons/{id}`, `/snt-members`, `/plots`, `/meters`. Всё фильтруется по `X-User-Id` и роли. |
| **v2 БД** | Модели + миграции `0001_init_v2`, `0002_meter_app_user`. Seed: 3 СНТ, 8 физ. лиц, члены, участки, plot_owners, пользователи (1 admin + 3 snt_user), счётчики (вода + электричество на участок). |
| **Авторизация** | `get_current_user` из `X-User-Id`. При отсутствии/невалидном — первый admin. Без паролей. |
| **Изоляция** | `snt_user` ограничен `user.snt_id` во всех нужных эндпоинтах. Фронт скрывает переключатель СНТ для `snt_user`. |
| **Frontend** | Разделы: Обзор, СНТ, Физ. лица, Члены СНТ, Участки, Счётчики. Переключатель темы, `localStorage` для пользователя. |

## В планах

- **Документы и проводки в v2:** В v2 нет `doc_accrual`, `doc_payment`, `reg_balance`. Начисления, платежи и логика регистров только в v1.
- **`charge_item` в v2:** Справочник статей начислений в v2 не добавлен.
- **CRUD во фронте:** Нет создания/редактирования/удаления лиц, участков, счётчиков и т.д.; только чтение.
- **Счётчики в модалке физ. лица:** В деталях показываются только членства и владение участками; счётчики по участкам не отображаются.
- **Настраиваемый API base:** Фронт использует жёстко прописанный `http://localhost:8001`.

---

## Известные ограничения

- **`X-User-Id` доверяемый:** Нет крипто/подписи; заголовок можно подделать для impersonation.
- **`run-v2.ps1`:** Рассчитан на Windows/PowerShell и `python` в PATH (или venv); на других ОС может потребоваться доработка.
- **v1:** Нужны PostgreSQL, миграции и seed (см. `DEV_LOG.md`); текущим фронтом не используется.
