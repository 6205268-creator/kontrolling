# Контекст проекта

## Краткая цель проекта

Система учёта для **СНТ** (садовые некоммерческие товарищества). Управляет товариществами (СНТ), участками, собственниками/членами, счётчиками и (в полной версии) начислениями и платежами. **v2** фокусируется на разделении **физ. лиц** и **членов СНТ**, владении участками, счётчиках и строгой изоляции данных по СНТ для не‑админов.

---

## Основные ограничения

- **Фронтенд не источник истины.** Все инварианты и расчёты — в backend/БД.
- **Мультитенантность по СНТ.** Во всех таблицах есть `snt_id`; данные разных СНТ не смешиваются.
- **Строгая изоляция для `snt_user`.** Пользователь, привязанный к одному СНТ, не должен видеть данные других СНТ.
- **Авторизация без паролей.** Только выбор пользователя в UI; идентификация по заголовку `X-User-Id`. Без сессий/токенов.
- **v1 (`backend_snt_core`) не трогаем.** v2 в `backend_snt_v2`; общий код не используем.

## Технологический стек

| Слой | v1 (core) | v2 (backend для UI) | Frontend |
|------|-----------|---------------------|----------|
| Runtime | Python | Python | Browser |
| Framework | FastAPI | FastAPI | Vanilla JS |
| DB | PostgreSQL | SQLite | — |
| ORM | SQLAlchemy | SQLAlchemy | — |
| Migrations | Alembic | Alembic | — |
| Config | pydantic-settings | pydantic-settings | — |
| UI | — | — | HTML, CSS, `fetch` |

- **Порты:** v1 API :8000, v2 API :8001, статический сервер фронта :3000.
- **Скрипты:** `run.ps1` для каждого backend; `run-v2.ps1` запускает v2 + фронтенд (PowerShell, Windows).
