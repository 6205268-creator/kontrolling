# План реализации v2: физ. лица, члены СНТ, участки

Перед реализацией выполнять **только** пункты из этого плана. Все изменения фиксировать в `CHANGELOG_IMPLEMENTATION.md`.

---

## Цели

- **Backend v2:** схема по `ARCHITECTURE_SNT_V2_PHYSICAL_PERSON.md`: `physical_person`, `snt`, `snt_member`, `plot`, `plot_owner`. Без meter, charge_item, документов — только справочники и связи.
- **Frontend:** переключатель СНТ (товариществ); при смене СНТ — режим «по выбранному СНТ». Разделы: **Физ. лица**, **Члены СНТ**, **Участки**. Клик по физ. лицу → показать связи: членства (в каких СНТ, периоды), участки (какие, периоды). Только просмотр + тестовые данные, без добавления/редактирования.
- **Оформление:** минималистичный современный стиль, простые иконки.

## Фаза 2. Backend v2

**Расположение:** отдельная папка `backend_snt_v2` (не трогаем `backend_snt_core`).

**СУБД:** SQLite (`snt_v2.db` в корне `backend_snt_v2`), чтобы не требовать PostgreSQL для демо.

| # | Действие | Детали |
|---|----------|--------|
| 2.1 | Структура проекта | `backend_snt_v2/`: `app/`, `alembic/`, `requirements.txt`, `alembic.ini`, `run.ps1` |
| 2.2 | Модели SQLAlchemy | `physical_person`, `snt`, `snt_member`, `plot`, `plot_owner` по v2-архитектуре |
| 2.3 | Alembic | Настроить под SQLite, одна миграция `0001_init_v2` создаёт все таблицы |
| 2.4 | Seed | Скрипт `scripts/seed.py`: 2–3 СНТ, 6–8 физ. лиц, членства, участки, владельцы. Хотя бы один физ. лицо — с несколькими участками и/или несколькими членствами |
| 2.5 | REST API | FastAPI. Эндпоинты: `GET /snts`, `GET /physical-persons`, `GET /snt-members?snt_id=`, `GET /plots?snt_id=`, `GET /physical-persons/{id}` (с `members` и `plot_owners`/участками). CORS для фронта |
| 2.6 | Запуск | `run.ps1`: миграции → seed → uvicorn на порту 8001 (чтобы не конфликтовать с 8000) |

## Фаза 3. Frontend

**Расположение:** существующая папка `frontend/`. Адаптировать под v2, без полной переделки.

| # | Действие | Детали |
|---|----------|--------|
| 3.1 | API-клиент | Базовый URL `http://localhost:8001`. Функции: загрузка СНТ, физ. лиц, членов по `snt_id`, участков по `snt_id`, физ. лица по `id` с связями |
| 3.2 | Переключатель СНТ | Выпадающий список в шапке; данные из `GET /snts`. При смене — перезагрузка членов и участков для выбранного СНТ |
| 3.3 | Навигация | Секции: **Обзор** (краткая сводка по выбранному СНТ), **Физ. лица**, **Члены СНТ**, **Участки**. Убрать или временно скрыть «Документы», «Отчёты» |
| 3.4 | Обзор | Карточки: кол-во физ. лиц (по СНТ или всего), членов, участков. Минимум текста |
| 3.5 | Физ. лица | Список (таблица или карточки). Клик по строке → открыть детальный вид (модалка или панель справа): ФИО, телефон, ИНН; блок «Членства» (СНТ, период); блок «Участки» (номер, СНТ, период) |
| 3.6 | Члены СНТ | Список, фильтр по выбранному СНТ. Колонки: физ. лицо, дата вступления, дата выхода. Только просмотр |
| 3.7 | Участки | Список по выбранному СНТ. Номер, при необходимости — текущий владелец (физ. лицо). Только просмотр |
| 3.8 | Стиль | Минимализм: нейтральная палитра, много воздуха, простые иконки (символы/ SVG). Сохранить переключатель светлая/тёмная тема |
