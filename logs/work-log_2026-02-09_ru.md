## Записи

### Запись 1 — 2026-02-09 (сессия по дизайну проекта, короткий перерыв)

**Что сделали:**

1. **Документ дизайна проекта** — Создали и обновляли `docs/project-design.md` как единый источник решений по дизайну.

2. **Стек технологий (зафиксирован):**
   - Бэкенд: Python, Django (REST API).
   - БД: PostgreSQL.
   - Фронт: React (SPA), Next.js пока не используем.
   - Архитектура: фронт и бэкенд раздельно, общение по REST API.

3. **Терминология и масштаб:** Система для **СТ** (садоводческое товарищество), законодательство **Беларусь** — не СНТ. Корректная аббревиатура: **СТ**.

4. **Развёртывание:** VPS в облаке; Docker в планах на потом (дальняя перспектива). Детализацию не делали.

5. **Авторизация:** Роли — администратор, казначей, председатель. Вход по логину и паролю. 2FA отложено.

6. **Мультитенантность:** Система рассчитана на **несколько товариществ (СТ)** в одной установке. В модели данных во всех основных таблицах должен быть ключ (tenant), разделяющий данные по кооперативу.

7. **Подход к модели данных:** Такой же, как к архитектуре — **Architecture as Code (AaC)**. Визуальное отображение связей между сущностями обязательно (диаграммы Mermaid в репозитории). Schema First. Как именно будем проектировать модель — договоримся отдельно.

8. **Правило стиля общения** — Добавлен `.cursor/rules/communication-style.mdc`: обращение на «ты», тон партнёров, без лишней формальности и без панибратства.

9. **Исследование:** Посмотрели существующие решения (1С:Садовод, SNT Club, open-source аналоги) и тренды по фронту (React/Next.js и Angular); зафиксировано в переписке, в project-design не выносили.
