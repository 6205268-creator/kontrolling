## Запись — 2026-02-14 (BPMN-вьювер и процесс оплаты)

**Что сделали:**

1. **BPMN-вьювер** (`docs/processes/bpmn-viewer.html`):
   - На базе bpmn-js (официальная библиотека bpmn.io)
   - Работает в браузере, без расширений Cursor
   - **Требует локальный HTTP-сервер** — file:// не грузит .bpmn (fetch)

   - **Двойной клик** по элементу → открыть панель, показать описание
   - **Одиночный клик** (когда панель открыта) → показать описание
   - Чекбокс «Показать панель описаний» — вкл/выкл вручную
   - Подсветка выбранного элемента (синий)
   - Documentation из `<bpmn:documentation>` отображается в панели

3. **Процесс оплаты** (`docs/processes/payments/process-payment-owner-account.bpmn`):
   - Два трека: Казначей (ручной) + Система (автоматический)

### Исправление «Скопировать XML» (2026-02-14)

**Проблема:** При экспорте через кнопку «Скопировать XML» в буфер обмена попадала неполная схема — при вставке в другой BPMN-редактор отображалась только часть диаграммы.

**Причина:** `getCurrentXml()` мог возвращать результат `viewer.saveXML()`, сериализация bpmn-js иногда теряла элементы (collaboration, участники).

**Решение:**
1. **Источник для копирования:** Введена `getXmlForCopy()` — всегда использует `lastLoadedXml` (содержимое исходного файла), без сериализации.
2. **Проверка полноты:** Перед копированием проверяется наличие collaboration, participant, нескольких процессов; при успехе показывается объём и пометка «схема с участниками».
3. **Fallback при ошибке:** Если `navigator.clipboard.writeText` недоступен (например, без HTTPS) — модальное окно с полным XML для ручного выделения и копирования.
