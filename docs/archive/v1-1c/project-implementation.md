# Controlling — План реализации (v2)

**Архитектура:** 1С:Enterprise (backend + БД) + OData + Vue 3 (frontend)  
**Принцип:** 1С — источник правды, Frontend — красивый интерфейс поверх OData.

---

## Как работать с этим документом

1. Найди первую незакрытую фичу (`- [ ]`).
2. Выполни все шаги, проверь результат.
3. После подтверждения — отметь `- [x]` и ✅ в заголовке.
4. Переходи к следующей.

---

## Фазы реализации

| Фаза | Содержание | Кто делает |
|------|-----------|------------|
| **0 — 1С** | Конфигурация, OData, разделитель | Ты (1С-разработчик) |

## Фаза 1 — Frontend: основа

### Фича 1: Инициализация проекта ✅/⬜

**Цель:** Создать структуру Vue 3 + TypeScript + PrimeVue + Vite.

**Шаги:**

- [ ] Создать проект: `cd frontend && pnpm create vite . --template vue-ts`
- [ ] Установить зависимости:
  ```
  pnpm install vue-router@4 pinia axios
  pnpm install primevue @primevue/themes primeicons
  pnpm install -D @types/node
  ```
- [ ] Создать структуру:
  ```
  frontend/src/
  ├── api/
  │   ├── odata.ts        # OData-клиент (базовый)
  │   └── endpoints/      # Методы по сущностям
  ├── stores/
  │   └── auth.ts
  ├── views/
  ├── components/
  ├── types/              # TypeScript-типы (1С сущности)
  ├── router/index.ts
  └── main.ts
  ```
- [ ] Настроить PrimeVue в `main.ts` (тема Aura или Lara)
- [ ] Настроить `vite.config.ts` с proxy на 1С OData:
  ```ts
  '/odata': { target: 'http://localhost/a/st', changeOrigin: true }
  '/a': { target: 'http://localhost', changeOrigin: true }
  ```
- [ ] Создать `frontend/.env.development`:
  ```
  VITE_1C_BASE_URL=/a/st
  VITE_ODATA_URL=/a/st/odata/standard.odata
  ```

**Проверка:**
- [ ] `pnpm dev` — сервер запускается на http://localhost:5173
- [ ] `pnpm build` — сборка без ошибок
- [ ] Proxy работает: запрос на `/odata/...` проксируется на 1С

### Фича 2: OData-клиент

**Цель:** Типизированный модуль для работы с 1С OData.

**Файлы:**

- `frontend/src/types/1c.ts` — типы сущностей 1С

**Шаги:**

- [ ] Создать `frontend/src/api/odata.ts`:

  - `ContributionType` — Вид взноса

## Фаза 5 — Деплой

### Фича 14: Docker + Nginx

**Цель:** Production-сборка и деплой на VPS.

**Файлы:**
- `frontend/Dockerfile`
- `nginx/nginx.conf`
- `docker-compose.yml`

**Шаги:**

- [ ] Создать `frontend/Dockerfile`:
  ```dockerfile
  FROM node:20-alpine AS builder
  WORKDIR /app
  COPY package.json pnpm-lock.yaml ./
  RUN npm install -g pnpm && pnpm install
  COPY . .
  RUN pnpm build

  FROM nginx:alpine
  COPY --from=builder /app/dist /usr/share/nginx/html
  COPY nginx/nginx.conf /etc/nginx/conf.d/default.conf
  ```

- [ ] Создать `nginx/nginx.conf`:
  - Раздача статики из `/usr/share/nginx/html`
  - Proxy `/a/` → 1С web-сервер (на хосте или в docker network)
  - HTTPS (сертификат Let's Encrypt или самоподписанный)
  - Заголовки CORS и кеширования

- [ ] Создать `docker-compose.yml` с сервисами:
  - `frontend` — nginx с собранным Vue-приложением

- [ ] Настроить переменные окружения для production:
  - `VITE_1C_BASE_URL` — URL 1С сервера на VPS

**Проверка:**
- [ ] `docker-compose up --build` — контейнер поднимается
- [ ] Открыть https://your-domain.com — приложение работает
- [ ] Запросы к `/a/` проксируются на 1С
