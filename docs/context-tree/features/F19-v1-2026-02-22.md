## Выполнено

**Версия:** 1
**Дата:** 2026-02-22

**Простыми словами:**
Реализованы два базовых отчёта для садоводческих товариществ: (1) отчёт по должникам — показывает все участки и счётчики с задолженностью, кто владеет и сколько должен; (2) отчёт о движении денег — сколько начислено, оплачено и потрачено за выбранный период, какой чистый остаток. Это позволяет председателю и казначею видеть финансовую картину СТ и контролировать должников.

**Роль в структуре проекта (технически):**
- Слой сервисов: `backend/app/services/report_service.py` — бизнес-логика отчётов, агрегация данных из Accrual, Payment, Expense, FinancialSubject, LandPlot, Meter
- Слой API: `backend/app/api/v1/reports.py` — REST endpoints с разграничением доступа (admin видит все СТ, treasurer/chairman — только своё)
- Слой схем: `backend/app/schemas/report.py` — Pydantic-схемы для валидации и сериализации ответов API
- Интеграция в main.py: router подключён к FastAPI приложению
- Тесты: `backend/tests/test_api/test_reports.py` — 11 тестов покрывают оба отчёта, фильтрацию, доступ, краевые случаи
- Зависимости: использует модели FinancialSubject (F04), Accrual/Payment (F05), Expense (F06), BalanceService (F14)
