# Фича 13: API endpoints — LandPlots (CRUD с PlotOwnership)

---
## Выполнено

**Версия:** 1
**Дата:** 2026-02-22

**Простыми словами:**
Реализован полный CRUD для земельных участков с поддержкой множественных владельцев. При создании участка автоматически создаётся финансовый субъект для будущих начислений и платежей. Можно добавлять и закрывать права собственности на участок (долевая собственность).

**Роль в структуре проекта (технически):**
- **Слой:** API (endpoints) + Service Layer
- **Зависимости:** F02 (Owner модель), F03 (LandPlot, PlotOwnership модели), F04 (FinancialSubject модель), F09 (схемы), F10 (аутентификация)
- **Интеграция:**
  - Использует модели `app.models.land_plot.LandPlot`, `app.models.plot_ownership.PlotOwnership`
  - Использует схемы `app.schemas.land_plot.LandPlotCreate/LandPlotUpdate/LandPlotWithOwners`
  - Использует схемы `app.schemas.plot_ownership.PlotOwnershipCreate/PlotOwnershipInDB`
  - Сервис `app.services.land_plot_service` — CRUD операции + автосоздание FinancialSubject
  - Подключен в `app.main.py` как `/api/v1/land-plots`
- **Что использует:** JWT-токены, разграничение по ролям (admin/treasurer)
- **Что от неё зависит:** F14/F15 (Accruals/Payments — начисления и платежи привязываются к FinancialSubject участка)

---

**Цель:** CRUD участков с владельцами, автосоздание FinancialSubject.

## Реализация

### Созданные файлы:

1. **`backend/app/schemas/plot_ownership.py`** — Pydantic схемы:
   - `PlotOwnershipBase` — owner_id, share_numerator, share_denominator, is_primary, valid_from, valid_to
   - `PlotOwnershipCreate` — для создания (land_plot_id опционально)
   - `PlotOwnershipUpdate` — для обновления
   - `PlotOwnershipInDB` — полная схема с ID и timestamps

2. **`backend/app/schemas/land_plot.py`** — обновлён:
   - `LandPlotCreate` — добавлено поле `ownerships: list[PlotOwnershipCreate]`
   - `LandPlotWithOwners` — схема ответа с владельцами и financial_subject

3. **`backend/app/services/land_plot_service.py`** — сервисный слой:
   - `create_land_plot()` — создание участка + FinancialSubject + ownerships
   - `get_land_plot()` — получение по ID
   - `get_land_plots_by_cooperative()` — список по СТ
   - `get_land_plot_with_owners()` — участок с владельцами и ФС
   - `update_land_plot()` — обновление
   - `delete_land_plot()` — удаление
   - `add_plot_ownership()` — добавить владельца
   - `close_plot_ownership()` — закрыть владение (valid_to)
   - `get_plot_ownership()` — получение владения по ID

4. **`backend/app/api/v1/land_plots.py`** — API endpoints:
   - `GET /api/v1/land-plots/` — список (фильтр по cooperative_id)
   - `GET /api/v1/land-plots/{id}` — получение с владельцами и ФС
   - `POST /api/v1/land-plots/` — создание с ownerships
   - `PATCH /api/v1/land-plots/{id}` — обновление
   - `DELETE /api/v1/land-plots/{id}` — удаление (admin)
   - `POST /api/v1/land-plots/{id}/ownerships` — добавить владельца
   - `PATCH /api/v1/land-plots/ownerships/{id}/close` — закрыть владение

5. **`backend/tests/test_api/test_land_plots.py`** — 13 тестов:
   - Создание с одним владельцем
   - Создание с несколькими владельцами (1/2, 1/2)
   - Получение списка
   - Получение по ID
   - Обновление
   - Удаление с проверкой прав
   - Добавление владельца
   - Закрытие владения
   - Проверка автосоздания FinancialSubject
