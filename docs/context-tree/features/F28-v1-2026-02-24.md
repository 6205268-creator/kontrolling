**Простыми словами:**  
На фронтенде сделана страница «Отчёт по должникам»: таблица с должниками (участок/счётчик, владелец, сумма долга), фильтр по минимальной сумме долга, сортировка по убыванию долга и экспорт в CSV. Для админа — выбор СТ, для председателя/бухгалтера — отчёт по своему СТ. Данные берутся из уже существующего API отчётов.

**Роль в структуре проекта (технически):**  
Store отчётов (`stores/reports.ts`) держит состояние отчёта по должникам и движения средств, дергает `GET reports/debtors` и `GET reports/cash-flow`. Типы `DebtorInfo` и `CashFlowReport` добавлены в `types/index.ts`. Роут `/reports/debtors` вложен в `/reports`; `ReportsView` — хаб со ссылкой на отчёт по должникам, вложенный контент рендерится через `<router-view />`. От этой фичи дальше опирается фича 29 (отчёт о движении средств уже использует тот же store).
