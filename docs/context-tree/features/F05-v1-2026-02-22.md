## Выполнено

**Версия:** 1
**Дата:** 2026-02-22

**Простыми словами:**
Созданы модели для финансовых операций: начисления (Accrual), платежи (Payment) и справочник видов взносов (ContributionType). Все финансовые операции идут через FinancialSubject — это позволяет единообразно работать с разными типами обязательств (участки, счётчики, решения собраний).

**Роль в структуре проекта (технически):**
Модели `Accrual`, `Payment`, `ContributionType` в `backend/app/models/`. Accrual имеет статусы created → applied → cancelled, Payment: confirmed → cancelled. Обе ссылаются на FinancialSubject (не напрямую на бизнес-объекты). ContributionType — справочник видов взносов (членский, целевой и т.д.). Миграция `0004_add_contribution_type_accrual_payment.py`. Тесты в `test_models/test_accrual.py` и `test_payment.py`. Основа для API начислений и платежей (фичи 14–16).
