# Фича 14: API endpoints — FinancialSubjects и балансы

---
## Выполнено

**Версия:** 1
**Дата:** 2026-02-22

**Простыми словами:**
Реализован просмотр финансовых субъектов и расчёт их балансов. Система автоматически считает задолженность: сумма применённых начислений минус сумма подтверждённых платежей. Можно получить баланс одного субъекта или всех субъектов товарищества сразу.

**Роль в структуре проекта (технически):**
- **Слой:** API (endpoints) + Service Layer
- **Зависимости:** F04 (FinancialSubject модель), F05 (Accrual, Payment модели), F10 (аутентификация)
- **Интеграция:**
  - Использует модели `app.models.financial_subject.FinancialSubject`, `app.models.accrual.Accrual`, `app.models.payment.Payment`
  - Использует схемы `app.schemas.balance.BalanceInfo`, `app.schemas.balance.FinancialSubjectInfo`
  - Сервис `app.services.balance_service` — расчёт балансов через SUM агрегацию
  - Подключен в `app.main.py` как `/api/v1/financial-subjects`
- **Что использует:** JWT-токены, разграничение по ролям (admin/treasurer)
- **Что от неё зависит:** F15 (Accruals — начисления влияют на баланс), F16 (Payments — платежи влияют на баланс), отчётность

## Реализация

### Созданные файлы:

1. **`backend/app/schemas/balance.py`** — Pydantic схемы:
   - `BalanceInfo` — финансовый субъект + total_accruals + total_payments + balance
   - `FinancialSubjectInfo` — информация о субъекте без баланса

2. **`backend/app/services/balance_service.py`** — сервисный слой:
   - `calculate_balance()` — расчёт баланса для одного субъекта
   - `get_balances_by_cooperative()` — массовый расчёт для всех субъектов СТ
   - `get_financial_subjects_by_cooperative()` — список субъектов по СТ
   - `get_financial_subject()` — получение субъекта по ID

3. **`backend/app/api/v1/financial_subjects.py`** — API endpoints:
   - `GET /api/v1/financial-subjects/` — список субъектов (фильтр по cooperative_id)
   - `GET /api/v1/financial-subjects/{id}/balance` — баланс конкретного субъекта
   - `GET /api/v1/financial-subjects/balances?cooperative_id={id}` — балансы всех субъектов СТ

4. **`backend/tests/test_services/test_balance_service.py`** — 7 тестов сервиса:
   - Простой баланс (одно начисление, один платёж)
   - Несколько начислений
   - Несколько платежей
   - Начисления со статусом 'created' не учитываются
   - Субъект не найден
   - Балансы по СТ (несколько субъектов)
   - Пустое СТ без субъектов

5. **`backend/tests/test_api/test_financial_subjects.py`** — 7 тестов API:
   - Получение списка субъектов
   - Получение баланса субъекта
   - Баланс не найден (404)
   - Балансы по СТ
   - Балансы по СТ для treasurer
   - Список без cooperative_id (auto для treasurer)
   - 403 при доступе к чужому СТ

## Формула расчёта

```
balance = total_accruals - total_payments

где:
- total_accruals = SUM(Accrual.amount) WHERE status = 'applied'
- total_payments = SUM(Payment.amount) WHERE status = 'confirmed'
```

**Положительный balance** = задолженность (начислено больше чем оплачено)
**Отрицательный balance** = переплата (оплачено больше чем начислено)
