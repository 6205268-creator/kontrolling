## Выполнено

**Версия:** 1  
**Дата:** 2026-02-24  

**Простыми словами:**  
Реализована форма создания начисления и экран списка начислений. Пользователь выбирает финансовый субъект (участок/счётчик и т.д.) и тип взноса, вводит сумму и период; начисление создаётся со статусом «создано». В списке для таких начислений доступны кнопки «Применить» и «Отменить». Добавлен справочник видов взносов в API для выбора типа в форме.

**Роль в структуре проекта (технически):**  
Frontend: Pinia store `accruals.ts` (fetch/create/apply/cancel), страницы AccrualsView (список с фильтрами по СТ и ФС) и AccrualCreateView (форма с выбором СТ, ФС, типа взноса, суммы и дат). Backend: роут `GET /api/v1/contribution-types/`, схема `ContributionTypeInDB`. Зависит от API начислений (F15) и навигации (F22). Дальше опирается на фича 27 (форма платежа) и отчёты по балансам.
