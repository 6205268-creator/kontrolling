## Выполнено

**Версия:** 1
**Дата:** 2026-02-22

**Простыми словами:**
Реализовано полное CRUD API для управления садоводческими товариществами (СТ). Admin может создавать, читать, обновлять и удалять любые СТ. Казначей и председатель видят только своё СТ. Сервисный слой отделён от API. Все 22 теста проходят.

**Роль в структуре проекта (технически):**
`backend/app/services/cooperative_service.py` — сервисный слой (create, get, get_cooperatives, update, delete). `backend/app/api/v1/cooperatives.py` — CRUD endpoints с разграничением доступа через `require_role(["admin"])` и фильтрацией по cooperative_id для не-admin пользователей. Service layer тестируется отдельно (11 тестов в `test_services/test_cooperative_service.py`), API тестируется с токенами (11 тестов в `test_api/test_cooperatives.py`). Зависимости: F02 (Cooperative модель), F09 (CooperativeInDB схема), F10 (auth deps).
