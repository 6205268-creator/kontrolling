## 🎯 Назначение параллельного проекта

### 1. Сбор требований

Каждое событие в сценариях (`scenarios.json`) содержит:
- **Требование к системе** → будущая фича в бэклоге
- **Статус реализации** → связь с реализованной функцией
- **Банковские выписки** → тестовые данные

**Пример:**
```json
{
  "id": 25,
  "title": "Платёж от третьего лица",
  "requirement": "Платёж «от третьего лица»: поле «плательщик» отдельно от «владелец участка»",
  "status": "not_implemented",
  "paymentDetails": {
    "isThirdParty": true
  }
}
```

→ Создаёт задачу в основном проекте: «Реализовать платёж от третьего лица»

### 2. Сценарное тестирование

Сценарии используются для:
- **E2E тестов** — автоматическое прохождение сценариев
- **Ручного тестирования** — проверка по шагам
- **Документирования** — как должно работать

### 3. Обратная связь от пользователей

Казначеи и председатели могут:
- Просматривать сценарии онлайн
- Оставлять рекомендации через форму
- Предлагать новые ситуации

## 📦 Структура данных

### Основной проект ← Параллельный проект

| Данные | Откуда | Куда |
|--------|--------|------|
| Сценарии (72 события) | `kontrolling-scenarios/src/data/scenarios.json` | Источник требований |
| Выписки ЕРИП (12 файлов) | `kontrolling-scenarios/public/data/statements/` | Тестовые данные |
| Статусы функций | Основной проект (API) | Отображение в сценариях |

## 🔄 Интеграция в будущем

### План интеграции

**Этап 1: Требования → Задачи**
- Автоматическая генерация GitHub Issues из сценариев
- Связь событий с task-трекером

**Этап 2: Тестирование**
- Генерация Playwright тестов из сценариев
- Валидация реализованных функций

**Этап 3: API интеграция**
- Сценарии загружают статусы из backend (`/api/scenario-statuses`)
- Backend использует сценарии для валидации

## 🚀 Развёртывание

### Основной проект
```bash
cd kontrolling
npm install
npm run dev
# http://localhost:5173
```

### Параллельный проект
```bash
cd kontrolling/kontrolling-scenarios
npm install
```

## 📊 Статусы событий

| Статус | Значение | Действие в основном проекте |
|--------|----------|----------------------------|
| ✅ Реализовано | Функция работает | Поддерживать, тестировать |
| ⚠️ Частично | Работает с ограничениями | Улучшать, дорабатывать |
| ❌ Не реализовано | Нет в системе | Создать задачу в бэклоге |
| 🔍 На пересмотр | Нужно решение | Обсудить с председателем |

## 📝 Обновление сценариев

При изменении `scenarios.json`:

1. **Обновить JSON** в `kontrolling-scenarios/src/data/scenarios.json`
2. **Закоммитить** в отдельный репозиторий
3. **Создать задачу** в основном проекте (если новое требование)
4. **Обновить статус** после реализации
