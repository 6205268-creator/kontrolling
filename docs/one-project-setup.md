# Один проект: только Vite (без старого dashboard на порту 80)

## Что происходит

- **http://127.0.0.1/** (порт 80) — отдаёт **старый** проект из **Docker** (контейнер с nginx).
- **http://127.0.0.1:5173/** — **новый** проект (Vite). После правки в `vite.config.ts` он слушает и по IPv4, так что этот адрес должен открываться.

Чтобы остался только один проект (новый на 5173), нужно перестать запускать старый.

## Вариант 1: Остановить контейнеры Docker (рекомендуется)

Старый фронт в этом репозитории — контейнер **kontrolling-frontend** (порт 80).

1. Останови только старый фронт (освободит порт 80):
   ```powershell
   docker stop kontrolling-frontend
   ```
2. Либо посмотри все контейнеры и останови нужный:
   ```powershell
   docker ps
   ```
   В колонке PORTS ищи `0.0.0.0:80->80/tcp`.
   ```powershell
   docker stop <имя_или_id_контейнера>
   ```
3. Либо останови все контейнеры проекта:
   ```powershell
   docker stop $(docker ps -q)
   ```
4. Если старый проект больше не нужен — образ и контейнер можно удалить (опционально):
   ```powershell
   docker ps -a
   docker rm kontrolling-frontend   # или другой id
   docker rmi <образ>   # если знаешь образ
   ```

После этого http://127.0.0.1/ перестанет открываться, а **http://127.0.0.1:5173/** будет единственным фронтом.

## Вариант 2: Не запускать Docker при старте Windows

Если не хочешь, чтобы что-то вообще слушало порт 80:

- **Docker Desktop**: Настройки → General → сними галку «Start Docker Desktop when you sign in», перезапусти Docker и останови контейнеры (как в варианте 1).
- Либо вручную не запускать контейнеры с пробросом 80.

## Как всегда работать с одним проектом

1. **Не** поднимать старый проект в Docker (порт 80).
2. Запускать только новый фронт (из корня или из `frontend/`):
   ```powershell
   npm run dev
   ```
   или из `frontend/`:
   ```powershell
   npm run dev
   ```
3. Открывать в браузере: **http://127.0.0.1:5173/** или **http://localhost:5173/**.

Бэкенд по желанию: из корня `npm run dev` поднимает и backend (8000), и frontend (5173); API тогда доступен через прокси с фронта по `/api`.
