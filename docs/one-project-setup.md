# Один способ запуска приложения

Раньше фронтенд мог открываться двумя путями (Docker на порту 80 и локальный Vite на 5173), из‑за чего возникала путаница со «старой» и «новой» версией.

Сейчас **единственная** точка входа для разработки:

1. **База и API:** Docker — `start.bat` или `docker compose up -d` (поднимает только db и backend).
2. **Фронтенд:** из корня проекта выполнить `npm run dev` и открыть в браузере **http://localhost:5173**.

Фронтенд в Docker больше не поднимается. Файлы `frontend/Dockerfile` и `frontend/nginx.conf` сохранены для возможной production-сборки (отдельно от docker-compose для разработки).

## Как работать

- Запустить контейнеры: `start.bat` (или `docker compose up -d`).
- В отдельном терминале из корня: `npm run dev`.
- Открыть: **http://localhost:5173** — логин по данным из `seed_db`/`seed_user`.
