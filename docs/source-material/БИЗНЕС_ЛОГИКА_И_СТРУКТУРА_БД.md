# БИЗНЕС-ЛОГИКА И СТРУКТУРА БАЗЫ ДАННЫХ
# Система учета хозяйственной деятельности садоводческих товариществ (СНТ)

## 1. ОБЩЕЕ ОПИСАНИЕ СИСТЕМЫ

### 1.1. Назначение
Система предназначена для автоматизации учета хозяйственной деятельности садоводческих некоммерческих товариществ (СНТ), включая:
- Учет участков и собственников
- Начисление взносов и платежей
- Учет показаний приборов учета
- Расчет пени и задолженностей
- Формирование платежных документов
- Расчет земельного налога
- Формирование отчетности

### 1.2. Основные возможности
- Начисление членских и целевых взносов
- Начисление по показаниям ИПУ (электроэнергия, вода, газ)
- Начисление по показаниям ОПУ (общие счетчики)
- Расчет и начисление пени
- Учет оплат (касса, банк, импорт из Клиент-Банка)
- Формирование платежных документов с QR-кодами
- Расчет земельного налога
- Различные формы отчетности

## 2. ЛОГИЧЕСКИЕ БЛОКИ СИСТЕМЫ

Система разбита на следующие логические блоки, каждый из которых может разрабатываться автономно:

### БЛОК 1: СПРАВОЧНИКИ (ЯДРО) ⭐
**Статус:** Минимальные сущности (ядро системы)

**Описание:** Базовые справочники, без которых система не может функционировать.

**Сущности:**
- Улицы
- Участки
- Владельцы
- Собственники участков
- Взносы и тарифы

**Связи:**
- Участки → Улицы (многие к одному)
- Собственники участков → Участки (многие к одному)
- Собственники участков → Владельцы (многие к одному)

**API интерфейсы:**
- `IStreetService` - работа с улицами
- `IAreaService` - работа с участками
- `IOwnerService` - работа с владельцами
- `IAreaOwnerService` - работа с собственниками участков
- `IContributionService` - работа со взносами

---

### БЛОК 2: ПРИБОРЫ УЧЕТА
**Статус:** Дополнительный блок

**Описание:** Учет индивидуальных (ИПУ) и общих (ОПУ) приборов учета.

**Сущности:**
- ИПУ (Индивидуальные приборы учета)
- ОПУ (Общие приборы учета)
- Показания ИПУ
- Показания ОПУ

**Связи:**
- ИПУ → Участки (многие к одному)
- ИПУ → ОПУ (многие к одному, опционально)
- Показания ИПУ → ИПУ (многие к одному)
- Показания ОПУ → ОПУ (многие к одному)

**API интерфейсы:**
- `IIPUService` - работа с ИПУ
- `IOPUService` - работа с ОПУ
- `IMeterReadingService` - работа с показаниями

**Зависимости:** Блок 1 (Справочники)

---

### БЛОК 3: НАЧИСЛЕНИЯ
**Статус:** Дополнительный блок

**Описание:** Расчет и учет начислений по взносам и услугам.

**Сущности:**
- Начисления
- Параметры расчета
- История расчетов

**Связи:**
- Начисления → Участки (многие к одному)
- Начисления → Взносы (многие к одному)
- Начисления → Собственники участков (многие к одному)
- Начисления → Показания ИПУ (многие к одному, опционально)

**API интерфейсы:**
- `IAccrualService` - расчет и учет начислений
- `ICalculationService` - сервис расчетов

**Зависимости:** Блок 1, Блок 2 (опционально)

---

### БЛОК 4: ОПЛАТЫ
**Статус:** Дополнительный блок

**Описание:** Учет поступлений оплат от собственников.

**Сущности:**
- Оплаты
- Приходные кассовые ордера (ПКО)
- Импорт оплат из банка

**Связи:**
- Оплаты → Участки (многие к одному)
- Оплаты → Взносы (многие к одному)
- Оплаты → Собственники участков (многие к одному)
- Оплаты → Начисления (многие к одному, опционально)

**API интерфейсы:**
- `IPaymentService` - работа с оплатами
- `IBankImportService` - импорт из банка

**Зависимости:** Блок 1, Блок 3 (опционально)

---

### БЛОК 5: ПЕНИ И ЗАДОЛЖЕННОСТИ
**Статус:** Дополнительный блок

**Описание:** Расчет пени и учет задолженностей.

**Сущности:**
- Настройки расчета пени
- Расчеты пени
- Задолженности

**Связи:**
- Расчеты пени → Участки (многие к одному)
- Расчеты пени → Взносы (многие к одному)
- Задолженности → Участки (многие к одному)

**API интерфейсы:**
- `IPenaltyService` - расчет пени
- `IDebtService` - работа с задолженностями

**Зависимости:** Блок 1, Блок 3, Блок 4

---

### БЛОК 6: ПЛАТЕЖНЫЕ ДОКУМЕНТЫ
**Статус:** Дополнительный блок

**Описание:** Формирование платежных документов (квитанций).

**Сущности:**
- Шаблоны платежных документов
- Сформированные платежные документы
- Настройки печати

**Связи:**
- Платежные документы → Участки (многие к одному)
- Платежные документы → Начисления (многие к одному)

**API интерфейсы:**
- `IPaymentDocumentService` - формирование ПД
- `ITemplateService` - работа с шаблонами

**Зависимости:** Блок 1, Блок 3

---

### БЛОК 7: ЗЕМЕЛЬНЫЙ НАЛОГ
**Статус:** Дополнительный блок

**Описание:** Расчет земельного налога.

**Сущности:**
- Расчеты земельного налога (неприватизированные участки)
- Расчеты земельного налога (ЗОП - земли общего пользования)

**Связи:**
- Расчеты ЗН → Участки (многие к одному)

**API интерфейсы:**
- `ILandTaxService` - расчет земельного налога

**Зависимости:** Блок 1

---

### БЛОК 8: ОТЧЕТНОСТЬ
**Статус:** Дополнительный блок

**Описание:** Формирование различных отчетов.

**Сущности:**
- Отчеты (виртуальные, формируются на лету)

**Типы отчетов:**
- Оборотная ведомость
- Реестр задолженностей
- Реестр начислений
- Реестр оплат
- Карточка участка
- Ведомость по показаниям счетчиков
- Реестр участков/собственников

**API интерфейсы:**
- `IReportService` - формирование отчетов

**Зависимости:** Все блоки

## 3. СТРУКТУРА БАЗЫ ДАННЫХ

### 3.1. ЯДРО СИСТЕМЫ (Минимальные сущности)

#### 3.1.1. Streets (Улицы)
```sql
CREATE TABLE Streets (
    Id INT PRIMARY KEY IDENTITY(1,1),
    Code NVARCHAR(50) NOT NULL UNIQUE,  -- Код улицы
    Name NVARCHAR(255) NOT NULL,        -- Название улицы
    CreatedAt DATETIME2 DEFAULT GETDATE(),
    UpdatedAt DATETIME2
);
```

#### 3.1.2. Areas (Участки)
```sql
CREATE TABLE Areas (
    Id INT PRIMARY KEY IDENTITY(1,1),
    StreetId INT NOT NULL FOREIGN KEY REFERENCES Streets(Id),
    Number NVARCHAR(50) NOT NULL,       -- Номер участка
    HouseNumber NVARCHAR(50),            -- Номер дома
    Area DECIMAL(10,2),                  -- Площадь участка
    CadastralNumber NVARCHAR(100),        -- Кадастровый номер
    IsPrivatized BIT DEFAULT 0,          -- Приватизировано
    IsUsed BIT DEFAULT 1,                -- Используется
    DistributionMethod NVARCHAR(50),     -- Метод распределения затрат ОПУ ЭЭ
    CreatedAt DATETIME2 DEFAULT GETDATE(),
    UpdatedAt DATETIME2,
    UNIQUE(StreetId, Number)
);
```

#### 3.1.3. Owners (Владельцы)
```sql
CREATE TABLE Owners (
    Id INT PRIMARY KEY IDENTITY(1,1),
    FullName NVARCHAR(255) NOT NULL,     -- Ф.И.О.
    RegistrationAddress NVARCHAR(500),   -- Адрес регистрации
    ActualAddress NVARCHAR(500),          -- Адрес фактический
    PostalAddress NVARCHAR(500),          -- Адрес почтовый
    BirthDate DATE,                       -- Дата рождения
    BirthPlace NVARCHAR(255),             -- Место рождения
    PassportData NVARCHAR(255),           -- Паспортные данные
    Phone NVARCHAR(50),                   -- Телефон
    MobilePhone1 NVARCHAR(50),            -- Моб. тел. 1
    MobilePhone2 NVARCHAR(50),            -- Моб. тел. 2
    Email NVARCHAR(255),                  -- E-mail
    AcceptanceDate DATE,                   -- Дата принятия
    ProtocolNumber NVARCHAR(50),          -- № протокола
    DepartureDate DATE,                   -- Дата выбытия
    CreatedAt DATETIME2 DEFAULT GETDATE(),
    UpdatedAt DATETIME2
);
```

#### 3.1.4. AreaOwners (Собственники участков)
```sql
CREATE TABLE AreaOwners (
    Id INT PRIMARY KEY IDENTITY(1,1),
    AreaId INT NOT NULL FOREIGN KEY REFERENCES Areas(Id),
    OwnerId INT NOT NULL FOREIGN KEY REFERENCES Owners(Id),
    FLSNumber NVARCHAR(50),               -- Номер ФЛС
    OwnershipShare DECIMAL(5,4) NOT NULL, -- Доля собственности (0-1)
    CertificateNumber NVARCHAR(100),      -- № свидетельства
    CertificateDate DATE,                  -- Дата рег. свидетельства
    AlienationDate DATE,                  -- Дата отчуждения
    IsInCalculation BIT DEFAULT 1,       -- Признак участия в расчете
    IsMember BIT DEFAULT 1,               -- Членство в СНТ
    IsCurrent BIT DEFAULT 0,              -- Текущий владелец
    CreatedAt DATETIME2 DEFAULT GETDATE(),
    UpdatedAt DATETIME2
);
```

#### 3.1.5. Contributions (Взносы)
```sql
CREATE TABLE Contributions (
    Id INT PRIMARY KEY IDENTITY(1,1),
    Code NVARCHAR(50) NOT NULL UNIQUE,    -- Код взноса
    FullName NVARCHAR(255) NOT NULL,      -- Наименование полное
    ShortName NVARCHAR(100),              -- Наименование краткое
    ContributionType NVARCHAR(50) NOT NULL, -- Тип взноса (Членские, Целевые, Электроэнергия, Вода, Газ, Отопление)
    CalculationMethod NVARCHAR(50) NOT NULL, -- Метод расчета (По площади, Фикс. взнос, По показаниям, По площади + фикс.)
    TariffCount INT DEFAULT 1,             -- Тарифность (для ЭЭ: 1, 2, 3)
    BankSearchTemplate NVARCHAR(500),     -- Шаблон поиска КБ
    CreatedAt DATETIME2 DEFAULT GETDATE(),
    UpdatedAt DATETIME2
);
```

#### 3.1.6. Tariffs (Тарифы)
```sql
CREATE TABLE Tariffs (
    Id INT PRIMARY KEY IDENTITY(1,1),
    ContributionId INT NOT NULL FOREIGN KEY REFERENCES Contributions(Id),
    TariffValue DECIMAL(10,4),            -- Тариф (для ЭЭ может быть несколько)
    TariffT1 DECIMAL(10,4),              -- Тариф Т1 (для ЭЭ)
    TariffT2 DECIMAL(10,4),              -- Тариф Т2 (для ЭЭ)
    TariffT3 DECIMAL(10,4),              -- Тариф Т3 (для ЭЭ)
    StartDate DATE NOT NULL,              -- Дата начала действия
    EndDate DATE,                         -- Дата окончания действия
    IsRural BIT DEFAULT 0,                -- Признак тарифа (сельский/обычный)
    CreatedAt DATETIME2 DEFAULT GETDATE()
);
```

#### 3.1.7. AreaContributions (Целевые взносы участков)
```sql
CREATE TABLE AreaContributions (
    Id INT PRIMARY KEY IDENTITY(1,1),
    AreaId INT NOT NULL FOREIGN KEY REFERENCES Areas(Id),
    ContributionId INT NOT NULL FOREIGN KEY REFERENCES Contributions(Id),
    StartDate DATE,
    EndDate DATE,
    CreatedAt DATETIME2 DEFAULT GETDATE()
);
```

### 3.2. ДОПОЛНИТЕЛЬНЫЕ СУЩНОСТИ

#### 3.2.1. IPU (Индивидуальные приборы учета)
```sql
CREATE TABLE IPU (
    Id INT PRIMARY KEY IDENTITY(1,1),
    AreaId INT NOT NULL FOREIGN KEY REFERENCES Areas(Id),
    Number NVARCHAR(100) NOT NULL,        -- Номер ИПУ
    Model NVARCHAR(255),                  -- Модель
    ServiceType NVARCHAR(50) NOT NULL,    -- Тип услуги (ЭЭ, Вода, Газ)
    TariffCount INT DEFAULT 1,            -- Тарифность
    InputDate DATE,                       -- Дата ввода
    OutputDate DATE,                      -- Дата вывода
    TariffType NVARCHAR(50),              -- Признак тарифа (Стандарт, Сельский)
    TariffStartDate DATE,                 -- Начало использования тарифа
    OPUId INT FOREIGN KEY REFERENCES OPU(Id), -- Общий счетчик
    Comment NVARCHAR(500),
    InitialReadingT1 DECIMAL(10,2),      -- Начальное показание Т1
    InitialReadingT2 DECIMAL(10,2),      -- Начальное показание Т2
    InitialReadingT3 DECIMAL(10,2),      -- Начальное показание Т3
    CreatedAt DATETIME2 DEFAULT GETDATE(),
    UpdatedAt DATETIME2,
    UNIQUE(AreaId, Number, ServiceType)
);
```

#### 3.2.2. IPUReadings (Показания ИПУ)
```sql
CREATE TABLE IPUReadings (
    Id INT PRIMARY KEY IDENTITY(1,1),
    IPUId INT NOT NULL FOREIGN KEY REFERENCES IPU(Id),
    ReadingDate DATE NOT NULL,           -- Дата показания
    ReadingT1 DECIMAL(10,2),              -- Показание Т1
    ReadingT2 DECIMAL(10,2),              -- Показание Т2
    ReadingT3 DECIMAL(10,2),              -- Показание Т3
    ConsumptionT1 DECIMAL(10,2),         -- Потребление Т1
    ConsumptionT2 DECIMAL(10,2),          -- Потребление Т2
    ConsumptionT3 DECIMAL(10,2),          -- Потребление Т3
    Losses DECIMAL(10,2),                 -- Потери
    CreatedAt DATETIME2 DEFAULT GETDATE(),
    UNIQUE(IPUId, ReadingDate)
);
```

#### 3.2.3. OPU (Общие приборы учета)
```sql
CREATE TABLE OPU (
    Id INT PRIMARY KEY IDENTITY(1,1),
    Number NVARCHAR(100) NOT NULL UNIQUE, -- Номер ОПУ
    Model NVARCHAR(255),                  -- Модель
    TariffCount INT DEFAULT 1,            -- Тарифность
    InputDate DATE,                       -- Дата ввода
    OutputDate DATE,                      -- Дата вывода
    CurrentTransformationRatio DECIMAL(10,4), -- Коэффициент трансформации тока
    LossPercentage DECIMAL(5,2),         -- Процент потерь в ЛЭП и ТТ
    IdleLossT1 DECIMAL(10,2),             -- Потери холостого хода Т1
    IdleLossT2 DECIMAL(10,2),            -- Потери холостого хода Т2
    IdleLossT3 DECIMAL(10,2),            -- Потери холостого хода Т3
    InitialReadingT1 DECIMAL(10,2),      -- Начальное показание Т1
    InitialReadingT2 DECIMAL(10,2),       -- Начальное показание Т2
    InitialReadingT3 DECIMAL(10,2),       -- Начальное показание Т3
    Comment NVARCHAR(500),
    CreatedAt DATETIME2 DEFAULT GETDATE(),
    UpdatedAt DATETIME2
);
```

#### 3.2.4. OPUReadings (Показания ОПУ)
```sql
CREATE TABLE OPUReadings (
    Id INT PRIMARY KEY IDENTITY(1,1),
    OPUId INT NOT NULL FOREIGN KEY REFERENCES OPU(Id),
    ReadingDate DATE NOT NULL,
    ReadingT1 DECIMAL(10,2),
    ReadingT2 DECIMAL(10,2),
    ReadingT3 DECIMAL(10,2),
    ConsumptionT1 DECIMAL(10,2),
    ConsumptionT2 DECIMAL(10,2),
    ConsumptionT3 DECIMAL(10,2),
    Losses DECIMAL(10,2),
    CreatedAt DATETIME2 DEFAULT GETDATE(),
    UNIQUE(OPUId, ReadingDate)
);
```

#### 3.2.5. Accruals (Начисления)
```sql
CREATE TABLE Accruals (
    Id INT PRIMARY KEY IDENTITY(1,1),
    AreaId INT NOT NULL FOREIGN KEY REFERENCES Areas(Id),
    AreaOwnerId INT NOT NULL FOREIGN KEY REFERENCES AreaOwners(Id),
    ContributionId INT NOT NULL FOREIGN KEY REFERENCES Contributions(Id),
    AccrualDate DATE NOT NULL,            -- Дата начисления
    PeriodStart DATE,                     -- Начало периода
    PeriodEnd DATE,                       -- Конец периода
    Amount DECIMAL(10,2) NOT NULL,        -- Сумма начисления
    IPUReadingId INT FOREIGN KEY REFERENCES IPUReadings(Id), -- Связь с показаниями
    Debt DECIMAL(10,2),                   -- Задолженность на дату
    Advance DECIMAL(10,2),                -- Аванс на дату
    Penalty DECIMAL(10,2),                -- Пени
    CreatedAt DATETIME2 DEFAULT GETDATE(),
    INDEX IX_Accruals_Area_Date (AreaId, AccrualDate),
    INDEX IX_Accruals_Contribution (ContributionId)
);
```

#### 3.2.6. Payments (Оплаты)
```sql
CREATE TABLE Payments (
    Id INT PRIMARY KEY IDENTITY(1,1),
    AreaId INT NOT NULL FOREIGN KEY REFERENCES Areas(Id),
    AreaOwnerId INT NOT NULL FOREIGN KEY REFERENCES AreaOwners(Id),
    ContributionId INT,                    -- NULL если оплата по нескольким статьям
    PaymentDate DATE NOT NULL,             -- Дата оплаты
    DocumentNumber NVARCHAR(50),           -- Номер документа
    Amount DECIMAL(10,2) NOT NULL,         -- Сумма оплаты
    PaymentType NVARCHAR(50),              -- Тип оплаты (Касса, Банк)
    Basis NVARCHAR(255),                  -- Основание
    CreatedAt DATETIME2 DEFAULT GETDATE(),
    INDEX IX_Payments_Area_Date (AreaId, PaymentDate)
);
```

#### 3.2.7. PaymentItems (Статьи оплаты)
```sql
CREATE TABLE PaymentItems (
    Id INT PRIMARY KEY IDENTITY(1,1),
    PaymentId INT NOT NULL FOREIGN KEY REFERENCES Payments(Id),
    ContributionId INT NOT NULL FOREIGN KEY REFERENCES Contributions(Id),
    Amount DECIMAL(10,2) NOT NULL,
    CreatedAt DATETIME2 DEFAULT GETDATE()
);
```

#### 3.2.8. Penalties (Пени)
```sql
CREATE TABLE Penalties (
    Id INT PRIMARY KEY IDENTITY(1,1),
    AreaId INT NOT NULL FOREIGN KEY REFERENCES Areas(Id),
    ContributionId INT NOT NULL FOREIGN KEY REFERENCES Contributions(Id),
    CalculationDate DATE NOT NULL,        -- Дата расчета
    DebtAmount DECIMAL(10,2),             -- Сумма задолженности
    DaysOverdue INT,                      -- Дней просрочки
    PenaltyAmount DECIMAL(10,2),          -- Сумма пени
    CreatedAt DATETIME2 DEFAULT GETDATE()
);
```

#### 3.2.9. PenaltySettings (Настройки расчета пени)
```sql
CREATE TABLE PenaltySettings (
    Id INT PRIMARY KEY IDENTITY(1,1),
    ContributionId INT FOREIGN KEY REFERENCES Contributions(Id), -- NULL для всех
    IsEnabled BIT DEFAULT 0,              -- Рассчитывать пени
    PenaltyPercent DECIMAL(5,2),         -- Процент пени
    StartDate DATE,                       -- Дата начала расчета
    CreatedAt DATETIME2 DEFAULT GETDATE(),
    UpdatedAt DATETIME2
);
```

#### 3.2.10. LandTaxCalculations (Расчеты земельного налога)
```sql
CREATE TABLE LandTaxCalculations (
    Id INT PRIMARY KEY IDENTITY(1,1),
    AreaId INT NOT NULL FOREIGN KEY REFERENCES Areas(Id),
    CalculationDate DATE NOT NULL,
    CalculationType NVARCHAR(50),         -- Неприватизированные, ЗОП
    CadastralValue DECIMAL(10,2),        -- Удельный показатель кадастровой стоимости
    TaxRate DECIMAL(5,2),                 -- Ставка налога %
    DeductionAmount DECIMAL(10,2),       -- Сумма вычета
    TaxAmount DECIMAL(10,2),            -- Сумма налога
    CreatedAt DATETIME2 DEFAULT GETDATE()
);
```

#### 3.2.11. PaymentDocuments (Платежные документы)
```sql
CREATE TABLE PaymentDocuments (
    Id INT PRIMARY KEY IDENTITY(1,1),
    AreaId INT NOT NULL FOREIGN KEY REFERENCES Areas(Id),
    DocumentDate DATE NOT NULL,
    DocumentType NVARCHAR(50),            -- Тип документа
    QRCode NVARCHAR(500),                 -- QR-код
    FilePath NVARCHAR(500),               -- Путь к файлу
    EmailSent BIT DEFAULT 0,              -- Отправлено на email
    CreatedAt DATETIME2 DEFAULT GETDATE()
);
```

#### 3.2.12. FLSNumbers (Финансовые лицевые счета)
```sql
CREATE TABLE FLSNumbers (
    Id INT PRIMARY KEY IDENTITY(1,1),

    FLSNumber NVARCHAR(50) NOT NULL,
    CreatedAt DATETIME2 DEFAULT GETDATE(),
```

## 6. ТЕХНИЧЕСКИЕ РЕКОМЕНДАЦИИ

### 6.1. Архитектура
- **Микросервисная архитектура** - каждый блок как отдельный сервис
- **API Gateway** - единая точка входа для всех сервисов
- **Event-Driven** - использование событий для связи между блоками
- **CQRS** - разделение команд и запросов для оптимизации

### 6.2. Технологический стек (рекомендации)
- **Backend:** .NET Core / Java Spring Boot
- **Database:** PostgreSQL / SQL Server
- **Cache:** Redis
- **Message Queue:** RabbitMQ / Apache Kafka
- **API:** REST / GraphQL

### 6.3. Безопасность
- Аутентификация и авторизация (JWT)
- Логирование всех операций
- Аудит изменений данных
- Резервное копирование

### 6.4. Производительность
- Индексы на часто используемых полях
- Кэширование справочников
- Пагинация для больших списков
- Асинхронная обработка расчетов
