# КОНЦЕПТУАЛЬНАЯ МОДЕЛЬ ДАННЫХ — МИНИМАЛЬНОЕ ЯДРО

Документ модели данных приведён в соответствие с минимальным ядром (Phase 1).  
Дальнейшие изменения вносить в этот файл и синхронизировать с `docs/data-model/entities-minimal.md` и `docs/data-model/schema-viewer.html`.

**Нормальная форма:** cooperative_id только у LandPlot. Связи «владелец ↔ СТ» нет — к товариществу идём через участок (LandPlot → Cooperative). Владение участком задаётся только через сущность PlotOwnership (участок, владелец, период, доля); участок может быть без владельца (нет записей PlotOwnership на нужную дату).

---

## 1. Ключевые сущности (ядро)

### 1.1 Садоводческое товарищество (Cooperative)

Одна запись = одно товарищество (СТ). Используется как tenant при мультитенантности.

Атрибуты:
- `id` (уникальный идентификатор) — ключ
- `name` (строка) — название товарищества

---

### 1.2 Владелец (Owner)

Абстрактная сущность: владелец участка или счётчика может быть **физическим лицом**, **юридическим лицом** (напр. исполком) или участок может быть **без владельца** (нет записей PlotOwnership на нужную дату). С СТ владелец напрямую не связан — к товариществу идём через участок (LandPlot → Cooperative). Тип владельца задаётся в owner_type.

Атрибуты:
- `id` (уникальный идентификатор) — ключ
- `owner_type` (строка: 'physical' — физ. лицо, 'legal' — юр. лицо, …) — тип владельца
- `name` (строка) — наименование (ФИО или название организации)
- `contact_phone` (строка) — телефон, опционально
- `contact_email` (строка) — электронная почта, опционально

Ограничения:
- Владелец не имеет прямой связи с СТ (cooperative_id нет).

---

### 1.3 Земельный участок (LandPlot)

Принадлежит одному СТ (cooperative_id). Владение участком задаётся только через сущность PlotOwnership (участок может быть без владельца — нет записей на нужную дату).

Атрибуты:
- `id` (уникальный идентификатор) — ключ
- `cooperative_id` (ссылка на Cooperative) — обязательно, внешний ключ
- `plot_number` (строка) — номер участка
- `area_sqm` (число) — площадь, кв. м
- `cadastral_number` (строка) — кадастровый номер, опционально
- `status` (строка: 'активный', 'зарезервирован', 'выведен' и т.д.) — статус участка

Ограничения:
- Принадлежность участка к СТ задаётся только здесь (cooperative_id).
- Связи участка с владельцами — только через PlotOwnership.

---

### 1.4 Право/доля собственности на участок (PlotOwnership)

Связь «участок ↔ владелец» с периодом действия и долей. По дате можно определить, кто в этот период владел участком и какой долей. Допускается несколько владельцев одного участка в один период (доли в сумме дают 1). Доля задаётся дробью: 1 = весь участок, 1/2 = половина, 1/3 = треть и т.д.

Атрибуты:
- `id` (уникальный идентификатор) — ключ
- `land_plot_id` (ссылка на LandPlot) — обязательно, внешний ключ
- `owner_id` (ссылка на Owner) — обязательно, внешний ключ
- `valid_from` (дата) — дата начала периода действия права
- `valid_to` (дата) — опционально; если пусто — право действует по сей день
- `share_numerator` (целое) — числитель доли (напр. 1)
- `share_denominator` (целое) — знаменатель доли (напр. 1 = вся доля, 2 = половина, 3 = треть)

Ограничения:
- Сумма долей (share_numerator/share_denominator) по активным на дату записям для одного участка обычно равна 1 (допускается меньше — часть участка без владельца).

---

### 1.5 Счётчик (Meter)

Прибор учёта (электроэнергия, вода, газ и т.д.). Привязан к владельцу (owner_id). Принадлежность к СТ — через участки владельца или контекст учёта.

Атрибуты:
- `id` (уникальный идентификатор) — ключ
- `owner_id` (ссылка на Owner) — обязательно, внешний ключ
- `number` (строка) — номер счётчика
- `meter_type` (строка) — тип: ЭЭ, вода, газ…
- `installation_date` (дата) — дата установки, опционально
- `status` (строка: 'в работе', 'выведен' и т.д.) — статус

---

## 2. Связи и кардинальности (ядро)

```
[Cooperative] —— (1:N) —— [LandPlot]
  (в СТ много участков)

[LandPlot] —— (1:N) —— [PlotOwnership] —— (N:1) —— [Owner]
  (у участка много записей права/доли; у владельца много записей по разным участкам и периодам;
   по дате определяем, кто владел участком и какой долей; участок может быть без владельца — нет записей на дату)

[Owner] —— (1:N) —— [Meter]
  (у владельца может быть несколько счётчиков)
```

Связи:
- LandPlot → Cooperative (cooperative_id) — участок принадлежит одному СТ
- PlotOwnership → LandPlot (land_plot_id) — право привязано к участку
- PlotOwnership → Owner (owner_id) — право привязано к владельцу; по valid_from/valid_to и доле — кто и какой долей владел в период
- Meter → Owner (owner_id) — счётчик привязан к владельцу
- Связи Owner → Cooperative нет (к СТ идём через LandPlot).

- Касса, банковские счета, банковские операции
- Собрания, решения собраний
- Показания счётчиков (MeterReading)
- Общее имущество СТ
- Первичные документы
