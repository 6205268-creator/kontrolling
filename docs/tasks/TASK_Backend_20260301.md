## Цель этапа

Привести код и документацию к единому источнику правды по размещению ORM-моделей (модули Clean Architecture), устранить прямую зависимость API от Infrastructure в deps и зафиксировать каноничность get_db и политику seed-скриптов.

## Контекст и ссылки

- **Затронутые файлы/папки**:  
  `backend/app/scripts/seed_db.py`, `backend/app/scripts/seed_user.py`,  
  `backend/app/api/deps.py`, `backend/app/modules/administration/`,

  `.cursor/rules/build/backend-testing-and-pitfalls.mdc`,  
  `frontend/e2e/conftest.py` (если e2e использует app.models)
- **Связанные дыры**: Источник правды для ORM-моделей (Critical), Границы слоёв API/Infrastructure (Critical), Каноничность get_db (High), Канонический seed (Medium).
- **Полезные ресурсы**: AGENTS.md (Module Structure, Pitfalls), docs/history/DECISIONS_LOG.md (DEC-001), docs/history/PENDING_GAPS.md.
- **История**: первый анализ, docs/history/ANALYSIS_LOG.md.

## Правила и стандарты для соблюдения

1. ORM-модели существуют только в `app/modules/{module}/infrastructure/models.py`. Регистрация для Alembic — только через `db/register_models.py`.
2. Presentation (api) не импортирует классы из infrastructure напрямую для типов, используемых в зависимостях (например текущий пользователь). Тип/интерфейс — в domain или application; маппинг — в infrastructure.
3. Единственная каноническая точка инъекции сессии БД в роутах — `app.api.deps.get_db`. Документировать в правилах тестирования, почему при необходимости переопределяются оба (deps.get_db и db_session.get_db) или привести к одному переопределению.
4. Любое изменение политики моделей или слоёв фиксируется в ADR или в обновлении AGENTS.md/PENDING_GAPS.

### Задача ID-Backend-001: Привести seed_db и seed_user к модульным моделям

**Описание**:  
Скрипты `seed_db.py` и `seed_user.py` сейчас импортируют из `app.models` (папка удалена). Заменить импорты на импорты из соответствующих модулей `app.modules.*.infrastructure.models` (или ввести тонкий фасад `app.models` как реэкспорт из модулей — по согласованию с архитектором). Сохранить идемпотентность seed_db.

**Зависимости**: Нет.  
**Критерии приемки (DoD)**:
- [x] `python -m app.scripts.seed_db` и `python -m app.scripts.seed_user` выполняются без ImportError.

- [x] В AGENTS.md или ADR зафиксировано: «Модели только в модулях; скрипты наполнения импортируют из модулей (или из фасада X)».

### Задача ID-Backend-002: Убрать прямую зависимость api/deps от infrastructure (тип пользователя)

**Описание**:

**Зависимости**: Нет.  
**Критерии приемки (DoD)**:

- [x] В документации (ADR или docs/architecture) зафиксировано правило: Presentation не зависит от Infrastructure.
