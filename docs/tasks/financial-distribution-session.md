## 🎯 Цель разработки

Создать **модуль распределения платежей** с настраиваемыми правилами приоритетов, который:
- Принимает платежи от физических лиц на лицевой счёт
- Автоматически распределяет по задолженностям согласно приоритетам
- Поддерживает частичное погашение
- Остаток средств хранится на лицевом счёте

---

## 📊 Принятые решения (зафиксировано)

### 1. Член СТ (Member)

**Решение:** Отдельная сущность с возможностью владения несколькими участками

```
Member (Член СТ):
  - id → Уникальный номер члена
  - owner_id → Владелец (физическое лицо, FK)
  - cooperative_id → Товарищество (СТ, FK)
  - status → Статус (active/expelled/resigned)
  - joined_date → Дата вступления

MemberPlot (Участки члена):
  - id → Уникальный номер
  - member_id → Член СТ (FK)
  - land_plot_id → Участок (FK)
  - share → Доля владения (например, 1/2)
```

---

### 2. Модуль распределения платежей

**Решение:** Комбинированный вариант (автоматически + ручное переопределение)

**Поток данных:**
```
Платёж → PersonalAccount (лицевой счёт) → Проверка задолженностей → 
Распределение по приоритетам → PaymentDistribution → Accrual (погашение)
```

**Принципы:**
- Распределение **только по задолженностям** (нет долга → не попадает)
- Остаток платежа → **на лицевом счёте** (не распределяется "впрок")
- Приоритеты настраивает пользователь

---

### 3. Приоритеты распределения

**Решение:** Настраиваемые приоритеты в модуле "НАСТРОЙКА"

**Пример:**
1. Членские взносы (приоритет 1)
2. Целевые взносы (приоритет 2)
3. Счётчики (вода/электричество, приоритет 3)

---

### 4. Модуль "НАСТРОЙКА"

**Решение:** Отдельный модуль с таблицами для масштабируемости

**Структура:**
```
SettingsModule:
  - id → Уникальный номер модуля
  - cooperative_id → Товарищество (FK)
  - module_name → "payment_distribution" / "contribution_types" / "meter_tariffs"
  - is_active → Активен/нет
  - created_at → Дата создания

PaymentDistributionRule:
  - id → Уникальный номер
  - settings_module_id → Модуль настроек (FK)
  - rule_type → "membership" / "land_plot" / "target" / "meter_water" / "meter_electricity"
  - priority → Число (1 = высший приоритет)
  - is_active → Активно/нет

ContributionTypeSettings:
  - id → Уникальный номер
  - settings_module_id → Модуль настроек (FK)
  - contribution_type_id → Вид взноса (FK)
  - default_amount → Сумма по умолчанию (опц.)
  - is_mandatory → Обязательный/нет
  - calculation_period → Период начисления (месяц/квартал/год)

MeterTariff:
  - id → Уникальный номер
  - settings_module_id → Модуль настроек (FK)
  - meter_type → "water" / "electricity" / "gas"
  - tariff_per_unit → Тариф за единицу (BYN)
  - valid_from → Дата начала действия
  - valid_to → Дата окончания (опц.)
```

---

### 5. Лицевой счёт (PersonalAccount)

**Решение:** Отдельная сущность с историей операций

**Структура:**
```
PersonalAccount:
  - id → Уникальный номер
  - owner_id → Владелец счёта (FK на Owner)
  - cooperative_id → Товарищество (FK на Cooperative)
  - balance → Текущий остаток (BYN)
  - status → "active" / "closed"
  - created_at → Дата открытия

PersonalAccountTransaction:
  - id → Уникальный номер
  - account_id → Лицевой счёт (FK)
  - payment_id → Платёж (FK, опц. для прихода)
  - distribution_id → Распределение (FK, опц. для расхода)
  - amount → Сумма (положительная для прихода, отрицательная для расхода)
  - type → "payment_received" / "distribution" / "refund" / "adjustment"
  - transaction_date → Дата операции
  - description → Описание (опц.)
```

---

### 6. Платёж и распределение

**Решение:** Один Payment + PaymentDistribution (отдельная таблица)

**Структура:**
```
Payment:
  - id → Уникальный номер
  - from_owner_id → Плательщик (FK на Owner)
  - total_amount → Общая сумма (BYN)
  - payment_date → Дата оплаты
  - document_number → Номер платёжного документа (опц.)
  - description → Описание (опц.)
  - status → "pending" / "distributed" / "partial" / "cancelled"
  - created_at → Дата создания

PaymentDistribution:
  - id → Уникальный номер
  - payment_id → Платёж (FK)
  - financial_subject_id → Финансовый субъект (FK)
  - amount → Распределённая сумма (BYN)
  - priority → Приоритет распределения (на момент распределения)
  - status → "pending" / "applied" / "partial" / "cancelled"
  - distributed_at → Дата распределения
```

## 🔄 Сценарии распределения

### Сценарий 1: Полное погашение (100 BYN = 100 BYN долга)

```
Платёж: 100 BYN
Задолженности:
  - Членство: 30 BYN (приоритет 1)
  - Участок: 50 BYN (приоритет 2)
  - Вода: 20 BYN (приоритет 3)

Результат:
  PaymentDistribution: 3 записи (30+50+20 = 100 BYN)
  PersonalAccount.balance = 0 BYN
  Все Accrual погашены полностью
```

---

### Сценарий 2: Частичное погашение (50 BYN платежа при 100 BYN долга)

```
Платёж: 50 BYN
Задолженности:
  - Членство: 30 BYN (приоритет 1) ✅
  - Участок: 50 BYN (приоритет 2) ⚠️
  - Вода: 20 BYN (приоритет 3) ❌

Результат:
  PaymentDistribution: 2 записи (30+20 = 50 BYN)
  PersonalAccount.balance = 0 BYN
  Accrual (членство) = погашено полностью
  Accrual (участок) = погашено 20 из 50 BYN (остаток 30 BYN)
  Accrual (вода) = не погашено (остаток 20 BYN)
```

---

### Сценарий 3: Платёж с остатком (150 BYN платежа при 100 BYN долга)

```
Платёж: 150 BYN
Задолженности: 100 BYN (все погашены)

Результат:
  PaymentDistribution: 3 записи (100 BYN распределено)
  PersonalAccount.balance = 50 BYN (остаток на счёте)
  Все Accrual погашены полностью
  Остаток 50 BYN ждёт будущих задолженностей
```
