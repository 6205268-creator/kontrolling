## Цель этапа

Установить единственный источник правды по технологическому стеку, ввести процесс ADR и индекс решений, зафиксировать политику глоссариев по доменам.

## Контекст и ссылки

- **Затронутые файлы/папки**:  
  `docs/project-design.md`,  
  `docs/architecture/adr/`,  
  `docs/architecture/glossary/`,  
  `.cursor/rules/architecture/`,  
  `AGENTS.md`
- **Связанные дыры**: Источник правды для стека (Critical), ADR индекс и процесс (High), Глоссарии по доменам (Medium).
- **Полезные ресурсы**: docs/history/DECISIONS_LOG.md (DEC-002), docs/history/PENDING_GAPS.md, AGENTS.md (Project Conventions).
- **История**: docs/history/ANALYSIS_LOG.md.

## Правила и стандарты для соблюдения

1. Канонический технологический стек и развёртывание описываются в AGENTS.md и при необходимости в docs/project-design.md; расхождение между ними недопустимо.
2. Каждое значимое архитектурное решение фиксируется в ADR; в репозитории ведётся индекс ADR (README или список в docs/architecture/adr/).
3. Глоссарий обязателен по каждому домену из модулей (cooperative, land, financial, accruals, payments, expenses, meters, reporting, administration); ответственный за актуализацию закреплён в правиле или в README глоссариев.

Убрать упоминания django-simple-history в пользу нейтральной формулировки (аудит/историзация — текущее решение или TBD).

### Задача ID-Architecture-003: Политика глоссариев по доменам

**Описание**:  
В `.cursor/rules/architecture/architecture-rules.mdc` (или в отдельном документе в docs/architecture/) зафиксировать: перечень доменов, для которых обязателен глоссарий (cooperative, land, financial, accruals, payments, expenses, meters, reporting, administration); путь — `docs/architecture/glossary/{domain}.md`; кто отвечает за создание и актуализацию (например «владелец домена» или «архитектор»). При возможности добавить заглушки или первый контент для 2–3 приоритетных доменов (например financial, accruals).

**Зависимости**: Нет.  
**Критерии приемки (DoD)**:
- Правило/документ существует и однозначно определяет перечень доменов и место глоссариев.
- Как минимум один домен помимо contributions имеет глоссарий или заглушку.

**Статус**: Выполнено (2026-03-01)
