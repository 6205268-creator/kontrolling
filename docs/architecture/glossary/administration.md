# Глоссарий домена «Администрирование (Administration)»

Термины, связанные с пользователями системы, ролями, доступом и аутентификацией.

| Термин | Определение | Пример использования в СТ |
|--------|-------------|----------------------------|
| **Пользователь (AppUser)** | Учётная запись для входа в систему: логин, пароль (хэш), имя, роль и привязка к СТ (опционально). Определяет, к каким данным и действиям пользователь имеет доступ. | Казначей СТ «Ромашка» — пользователь с role=treasurer, cooperative_id=«Ромашка». |
| **Роль (role)** | Тип доступа пользователя: admin (администратор), chairman (председатель), treasurer (казначей). Роль задаёт набор разрешённых операций и видимость данных. | treasurer — начисления, платежи, расходы своего СТ; chairman — только просмотр и отчёты. |
| **Администратор (admin)** | Пользователь с ролью admin: полный доступ к настройкам и данным всех СТ; может выполнять функции председателя и казначея для любого товарищества. | Администратор настраивает новые СТ и пользователей, просматривает отчёты по всем СТ. |
| **Председатель (chairman)** | Пользователь с ролью chairman: просмотр данных своего СТ, отчётность, контроль долгов. Не может создавать или изменять начисления, платежи и расходы. | Председатель формирует отчёт о должниках и отчёт о движении средств за квартал. |
| **Казначей (treasurer)** | Пользователь с ролью treasurer: операционная работа по своему СТ — регистрация начислений, платежей и расходов. Видит только данные своего кооператива. | Казначей вносит платёж 150 BYN на участок № 12, создаёт расход на ремонт дороги. |
| **Контекст кооператива пользователя (cooperative_id)** | Идентификатор СТ, к которому привязан пользователь (кроме admin). Все запросы к данным фильтруются по этому контексту. | У казначея cooperative_id = «Ромашка» — он видит только участки и платежи этого СТ. |
| **Аутентификация** | Проверка личности при входе: логин и пароль; выдаётся JWT-токен. Пароль хранится в виде хэша (bcrypt). | Пользователь вводит логин и пароль → система выдаёт токен для доступа к API. |

Смежные термины: кооператив и мультитенантность — [cooperative.md](cooperative.md).
