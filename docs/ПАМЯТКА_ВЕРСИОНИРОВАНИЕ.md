## Ежедневный цикл
### Перед началом работы
```powershell
cd e:\PROGECTS\kontrolling
git pull origin main
```
### После изменений
```powershell
git status
git add <файлы или .>
git commit -m "Краткое описание"
git push origin main
```
**Правило:** одна логическая задача — один коммит.

## Что под версией (коммитить)
- Весь код: `backend/`, `frontend/`
- Документация: `docs/`
- Правила и планы: `.cursor/rules/`, `docs/plan/`
- CI и владельцы: `.github/`
**Не коммитить:** `.env`, `node_modules/`, `__pycache__/`, `.vscode/` (уже в `.gitignore`).
---
## Что проверяется на GitHub автоматически
При каждом **push** в `main`:
- **Backend:** тесты (`pytest`), линтер (`ruff`), архитектурный линтер.
- **Frontend:** линт и тесты (`pnpm run lint`, `pnpm run test`).
Проверки смотри в **Actions** на GitHub: зелёная галочка — всё ок.
