# BPMN процессы в проекте КОНТРОЛЛИНГ

## Обзор

В проекте используются **BPMN 2.0** диаграммы для моделирования бизнес-процессов. Это дополняет C4 архитектурные диаграммы (Mermaid), давая детальное представление о workflow.

## Инструменты

- **Вьювер:** `docs/processes/bpmn-viewer.html` (на базе bpmn-js от Camunda)
- **Редактор:** Любой BPMN 2.0 редактор (рекомендуется Camunda Modeler)
- **Формат:** BPMN 2.0 XML

## Структура

```
/docs/processes/
  bpmn-viewer.html           — HTML вьювер для просмотра всех диаграмм
  /payments/                 — процессы оплаты
    process-payment-owner-account.bpmn
    _history/                — backup (создаётся вручную)
  /accruals/                 — процессы начисления
  /reconciliation/           — процессы сверки с банком
```

## Реализованные процессы

### 1. Оплата через лицевой счёт владельца
**Путь:** `docs/processes/payments/process-payment-owner-account.bpmn`

**Участники:**
- **Банк** (внешняя система)
- **Казначей СТ** (ручной процесс)
- **Система** (автоматический процесс)

**Описание:**

#### Трек 1: Казначей (ручной процесс)
1. Получена банковская квитанция (от банка)
2. Открыть квитанцию
3. Проверить сумму поступления
4. Определить владельца участка (физ. лицо)
5. Зачислить деньги на лицевой счёт владельца (Service Task)
6. Деньги зачислены → триггер для системы

#### Трек 2: Система (автоматический, каждую минуту)
1. Таймер запускает процесс каждую минуту
2. Проверить все лицевые счета владельцев
3. **Развилка:** Есть остаток на счёте?
   - Нет → конец цикла
   - Да → продолжить
4. Получить список задолженностей владельца (Accrual)
5. **Развилка:** Есть задолженности?
   - Нет → конец цикла
   - Да → списание по приоритетам
6. **Оплатить электричество** (приоритет 1)
7. **Оплатить членские взносы** (приоритет 2)
8. **Оплатить целевые взносы** (приоритет 3)
9. Обновить баланс участка (Начисления − Платежи)
10. Цикл завершён

**Концепция:**
- Деньги поступают на **лицевой счёт владельца** (OwnerAccount)
- Это промежуточный буфер, не привязан к конкретным начислениям
- Система автоматически списывает деньги по приоритетам

## Связь с моделью данных

### Новая сущность (требуется добавить):

**OwnerAccount** (Лицевой счёт владельца)
```
Attributes:
- owner_id (FK → Owner)
- balance (остаток, BYN)
- last_updated (дата последнего изменения)
```

**Связь с Payment:**
- Когда казначей зачисляет деньги → создаётся Payment + увеличивается OwnerAccount.balance
- Автоматическое списание создаёт Payment для каждого погашенного Accrual

## Как посмотреть диаграмму

### ⚠️ Важно для ассистента

**BPMN-вьювер требует локальный HTTP-сервер** (file:// не грузит .bpmn). Для показа в браузере:
1. Запустить: `cd docs/processes && python server.py` (рекомендуется) или `python -m http.server 8888`
2. Открыть: http://localhost:8888/bpmn-viewer.html

### Вариант 1: Через server.py (рекомендуется — с сохранением в проект)
```powershell
cd docs/processes
python server.py
```
Открыть: http://localhost:8888/bpmn-viewer.html

**Кнопка «Сохранить в проект»** — сохранит текущую диаграмму в файл проекта (например `payments/process-payment-owner-account.bpmn`). Работает только при запуске через `server.py`.

### Вариант 2: Через простой веб-сервер (без сохранения)
```powershell
cd docs/processes
python -m http.server 8888
```
Открыть: http://localhost:8888/bpmn-viewer.html — кнопка «Сохранить в проект» выдаст подсказку запустить `server.py`.

### Вариант 3: Через браузер напрямую
Двойной клик по `docs/processes/bpmn-viewer.html` — file:// может не загрузить .bpmn.

### Экспорт и импорт
- **Сохранить в проект** — записать диаграмму в файл проекта (требует `server.py`)
- **Скачать .bpmn** — сохранить текущую диаграмму как файл (положить в проект вручную)
- **Скачать как backup** — скачать с timestamp в имени (например `process-payment-owner-account_20260214_143022.bpmn`) для сохранения в `_history`
- **Скопировать XML** — скопировать BPMN XML в буфер обмена
- **Загрузить файл** — выбрать .bpmn с диска
- **Вставить XML** — вставить XML из буфера (модальное окно)

### Версионирование
- **До 5 версий в сессии** — при каждом импорте (загрузка, вставка) предыдущая версия сохраняется в `sessionStorage`
- **Откатить** — восстановить последнюю сохранённую версию
- **Папка _history** — для долгосрочных backup создаётся вручную. Рекомендуется хранить до 5 файлов с timestamp. Откат из _history — через «Загрузить файл» и выбор нужного backup

### Интерактивность
- **Zoom:** колёсико мыши над диаграммой (без Ctrl)
- **Перемещение:** зажать ЛКМ и двигать
- **Панель описаний:** скрыта по умолчанию
  - Двойной клик → открыть панель
  - Одиночный клик (когда открыта) → показать описание
  - Чекбокс справа → вкл/выкл панель
- **Подсветка:** выбранный элемент светится синим
