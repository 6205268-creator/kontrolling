СТРОГИЕ ЗАПРЕТЫ:
- Не придумывать бизнес-правила: порядок погашения взносов, пени, состав статей, формулы тарифов — только из документов владельца, ADR или явного ТЗ модуля.
- Не описывать и не вызывать несуществующие эндпоинты: сверяйся с backend/app/modules/**/api и app/main.py.
- Не переиспользовать EVT-ID под новое событие: новый кейс = новый номер (EVT-025…).
- Раннер (Playwright, субтитры, CI) — Фаза B ТЗ; начинай только если пользователь явно попросил реализовать автоматизацию.

- Добавлять строки по замечаниям казначея/владельца с колонкой «Утверждение»: черновик | утверждено для прогона.
- Строки с тегом smoke в матрице — кандидаты для будущего короткого прогона.

ЕСЛИ ПОПРОСЯТ ФАЗУ B:
- Реализуй по §7 ТЗ: профили validate, smoke, full, api, interactive, gaps; HTTP как у пользователя; рутина допускается через API; UI-критика через Playwright; субтитры отдельным мини-приложением, не внутри Vue Controlling; артефакты summary.json + steps.jsonl + скриншоты; чистая БД перед полным прогоном — согласовать с pytest/seed в проекте.
