# Сводка переговоров: сценарий «год жизни СТ» + раннер (для анализа другим агентом)

**Дата сводки:** 2026-03-20  
**Аудитория:** агент или человек, которому нужен контекст без чтения всей переписки.

---

## 1. Замысел владельца

- Нужна **имитация реальной жизни** садового товарищества (~10 участков, вымышленные персонажи) за **календарный год (январь–декабрь)**.
- Цели: **демонстрация** продукта Controlling, **проверка пробелов** в функционале («дыры»), в перспективе — **интерактивное обучение** казначея.
- **Распределение платежей по статьям** на первых порах — **вручную** в UI; автоматическое распределение — отложено.
- **Один владелец на участок** в сценарии; суммы в **BYN**.
- Желательно: прогон **как пользователь** (**HTTP/API** + для важных мест **браузер/Playwright**); **рутина** (например массовое создание участков) — не десятки кликов, а API/сид + комментарий «открой список и проверь».
- **Субтитры/описание шага** — **отдельно** от UI Controlling (не ломать продукт).
- Один **источник правды** по событиям: **одна таблица-матрица**, без дублирования «второго сценария»; казначей может дополнять — **новые строки = новые EVT-ID**.
- Ветка Git для сценария: `docs/scenario-life-year-matrix`; в **main** не коммитить/не мержить без команды владельца.

## 3. Недопонимание с промптом (важно для анализа)

- Пользователь вставил **только** текст промпта **v2.0** без явной задачи.
- В v2.0 в конце было: *«если нет задачи — спроси»* — агент **корректно** предложил варианты (аудит / новые EVT / раннер).
- Одновременно в промпте было: раннер только по явной просьбе — без фразы «сделай скрипт» агент **не** начинал Фазу B.
- **Исправление в v2.1:** сверху пояснение для пользователя; дефолт без задачи = **сразу аудит матрицы**; готовые однострочные «Задача: …».
