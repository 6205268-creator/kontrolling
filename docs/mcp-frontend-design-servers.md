# MCP и фронтенд-дизайн (под стек проекта, без Figma)

Список MCP, полезных для вёрстки и UI в этом проекте. Figma не используем — опора на дизайн-систему в коде и документацию библиотек.

---

## Стек проекта (фронтенд)

| Технология | Версия / детали |
|-------------|------------------|
| **Vue** | 3.5 |
| **TypeScript** | 5.9 |
| **Vite** | 7 |
| **Pinia** | состояние |
| **Vue Router** | маршруты |
| **Стили** | один глобальный `frontend/src/style.css` — **CSS-переменные** (дизайн-система), без Tailwind |
| **Шрифт** | Manrope (Google Fonts) |
| **Тема** | светлая/тёмная через `data-theme`, палитра «сад/СТ» (зелёный #10b981 и др.) |
| **Иконки** | lucide-vue-next |
| **Уведомления** | vue-sonner |
| **Графики** | Chart.js + vue-chartjs |
| **E2E** | Playwright |
| **Юнит-тесты** | Vitest |
| **UI-библиотека** | нет (свои компоненты на базе `style.css`) |

Figma в проекте нет — дизайн-контекст задаётся через `frontend/src/style.css` (переменные, типографика, цвета, отступы).

## Что добавить для «нормального» фронтенд-дизайна (без Figma)

### 1. cursor-ide-browser (в приоритете)

- Живой браузер: открыть `http://localhost:5173`, скриншоты, проверка вёрстки.
- AI может смотреть результат и предлагать правки по факту, а не «вслепую».
- В Cursor уже доступен в списке MCP — достаточно добавить в `.cursor/mcp.json` (конфиг смотреть в настройках MCP).

### 2. Опора на дизайн-систему в коде

Чтобы AI не выдумывал цвета и отступы:

- **Единый источник правды:** `frontend/src/style.css` — все переменные (`--color-*`, `--text-*`, `--font-*` и т.д.).
- Имеет смысл завести правило для агентов: при любой вёрстке/стилизации во фронте использовать только эти переменные и не вводить новые хардкод-цвета/шрифты. Пример формулировки для Cursor rule: «При правках Vue-компонентов и стилей использовать только переменные из `frontend/src/style.css`; не добавлять новые цвета/шрифты без согласования.»

### 3. Playwright MCP (по желанию)

- В проекте уже есть Playwright для E2E.
- MCP-сервер Playwright из [modelcontextprotocol/servers](https://github.com/modelcontextprotocol/servers) даёт AI возможность запускать браузер, делать скриншоты, проверять страницы — полезно для регресса и «как это выглядит».
- Добавлять только если нужна автоматическая проверка UI из чата/агента.

### 4. Графики (Chart.js)

- Графики уже на Chart.js + vue-chartjs.
- Context7 подтягивает актуальную документацию Chart.js — для дизайна графиков и осей этого достаточно. Отдельные MCP (ECharts, Vega-Lite и т.д.) имеют смысл только при смене стека визуализации.

## Чего не подключать в твоём случае

- **Figma MCP** — Figma не используем.
- **Storybook MCP** — Storybook в проекте нет; подключать только если решишь завести Storybook.
- **Shadcn/PrimeVue/Vuetify MCP** — у нас Vue и своя дизайн-система в CSS; эти MCP заточены под свои библиотеки. Shadcn — под React.

---

## Рекомендуемый порядок действий

1. **Проверить cursor-ide-browser** — включён ли в `.cursor/mcp.json`. Если да — использовать для проверки экранов после правок.
2. **Добавить правило про `style.css`** — при задачах по фронту и дизайну указывать AI использовать только переменные из `frontend/src/style.css`.
3. **Shadcn-ui MCP** — оставить только если полезен как источник идей; если AI начинает предлагать React/Shadcn-код вместо Vue — отключить.
4. **Playwright MCP** — по желанию, если нужны скриншоты и проверка UI из чата.
