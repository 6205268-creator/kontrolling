# Product Context

## 🎯 Почему существует этот проект

### Проблема

Садоводческие товарищества (СТ) сталкиваются с типичными проблемами:

1. **Финансовая непрозрачность**
   - Владельцы не знают, куда идут взносы
   - Казначеи тратят часы на сведение баланса в Excel
   - Задолженности обнаруживаются слишком поздно

2. **Ручной учёт**
   - Показания счётчиков передаются в чатах/бумажках
   - Начисления считаются вручную по формулам
   - Платежи разносятся «на глазок»

3. **Отсутствие истории**
   - При смене казначея теряются данные
   - Невозможно восстановить «кто когда заплатил»
   - Решения общих собраний не фиксируются

4. **Должники**
   - Нет автоматических напоминаний
   - Сложно отследить динамику задолженности
   - Приходится «бегать за каждым»

---

## 💡 Решение

**КОНТРОЛЛИНГ** — это единая система для:

### Для владельцев

- ✅ **Прозрачность** — видеть все начисления и платежи в личном кабинете
- ✅ **Контроль** — отслеживать баланс в реальном времени
- ✅ **Удобство** — передача показаний счётчиков онлайн

### Для казначея

- ✅ **Автоматизация** — массовое создание начислений по формулам
- ✅ **Учёт** — все платежи привязаны к лицевым счетам
- ✅ **Отчёты** — должники и cash flow в один клик
- ✅ **История** — полная аудитория всех операций

### Для председателя

- ✅ **Контроль** — мониторинг финансовых показателей
- ✅ **Планирование** — понимание, сколько собрано и сколько предстоит
- ✅ **Решения** — фиксация решений общих собраний в системе

---

## 🎯 Проблемы, которые решаем

| Проблема | Решение |
|----------|---------|
| **Excel-таблицы** | Единая база данных с доступом 24/7 |
| **Бумажные квитанции** | Электронные уведомления в личном кабинете |
| **Потеря истории** | Полная аудитория всех изменений |
| **Ручные начисления** | Автоматический расчёт по формулам |
| **Должники** | Отчёт по должникам с фильтрами |
| **Счётчики** | История показаний с графиками |
| **Разные форматы** | Стандартизированные отчёты и документы |

## 📈 Цели проекта

### Краткосрочные (3-6 месяцев)

- [ ] Полная автоматизация начислений и платежей
- [ ] Интеграция с SMS/Email для уведомлений
- [ ] Мобильная версия для владельцев

### Долгосрочные (6-12 месяцев)

- [ ] Онлайн-оплата через эквайринг
- [ ] Интеграция с банками (импорт выписок)
- [ ] Голосования на общих собраниях онлайн

## 🎭 Акторы (пользователи системы)

### Person_Sobstvennik (Владелец)

**Цели**:
- Видеть баланс своего участка
- Передавать показания счётчиков
- Оплачивать взносы онлайн

**Боли**:
- Не понимает, из чего складывается сумма
- Забывает сроки оплаты
- Теряет квитанции

---

### Person_Kaznachey (Казначей)

**Цели**:
- Вести учёт всех финансовых операций
- Формировать отчёты для председателя
- Контролировать задолженности

**Боли**:
- Тратит часы на сведение таблиц
- Ошибается в ручных расчётах
- Не видит полной картины

---

### Person_Predsedatel (Председатель)

**Цели**:
- Контролировать финансы СТ
- Планировать бюджет
- Отчитываться перед членами СТ

**Боли**:
- Не доверяет Excel-отчётам
- Не видит реальную картину
- Тратит время на «выбивание» долгов

---

### System_Bank (Внешняя система)

**Роль**: Банковские выписки, онлайн-платежи

**Взаимодействие**:
- Импорт платёжек из банка
- Экспорт платёжных поручений
