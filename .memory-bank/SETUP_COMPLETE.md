## 🚀 Как использовать

### Вариант 1: Автоматически (через MCP)

**Cursor откроет Memory Bank автоматически** при начале диалога.

**Команды** (если MCP сервер подключён):

```
/memory-bank read activeContext
/memory-bank read progress
/memory-bank update progress
```

---

### Вариант 2: Вручную (рекомендуется для начала)

**Перед началом сессии**:

1. Открой `.memory-bank/activeContext.md`
2. Прочитай раздел «Текущие задачи»
3. Начни диалог с контекстом:

> «Привет! Смотри activeContext.md — сейчас работаем над [задача].
> Нужно [конкретная просьба].»

**После сессии**:

1. Обнови `.memory-bank/activeContext.md`:
   - Что сделано
   - Что планируется дальше

2. Обнови `.memory-bank/progress.md`:
   - Добавь запись в историю
   - Обнови метрики

### Cursor Rules

Memory Bank дополняет **Cursor Rules**:

| Система | Назначение |

| **Cursor Rules** | Как работать (style, workflow, conventions) |

## 🛠️ Troubleshooting

### MCP не работает

**Проверь**:

```bash
# Установлен ли пакет
pip show mcp_memory_bank

# Конфигурация
cat .cursor/mcp.json
```

**Решение**:

```json
{
  "mcpServers": {
    "memory-bank": {
      "command": "python",
      "args": ["-m", "mcp_memory_bank"]
    }
  }
}
```

---

### Документы устаревают

**Проблема**: `activeContext.md` не обновляется

**Решение**:
1. Поставь напоминание (после каждой сессии)
2. Используй шаблон (см. выше)
3. Автоматизируй через MCP (если работает)
